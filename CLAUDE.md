## Стек (не менять без явного запроса)

- Backend: FastAPI + asyncpg, миграции — Alembic.
- Frontend: Next.js 14 App Router + TypeScript + Tailwind.
- DB: PostgreSQL. Без ORM с магией (не тащить SQLAlchemy ORM-слой поверх asyncpg без запроса) —
  Абас пишет raw SQL / query builder уровня asyncpg, ему так удобнее и быстрее ревьюить.
- Auth: JWT access+refresh, argon2/bcrypt для паролей команды.
- Bot: aiogram3, отдельный процесс, общается с CRM только через внутренний REST API
  (никогда напрямую в БД CRM из бота).
- Деплой: Hetzner VPS, отдельный PM2-процесс от aisolution.uz, отдельная БД.

## Hard constraints (см. CRM_SPEC.md раздел 1) — нарушение = баг, не "оптимизация"

1. Один лид = один owner. Пока owner не назначен — общая очередь, видна всем, изменять
   может только founder. После назначения — только owner или founder.
2. Soft delete везде. Никаких `DELETE FROM` в бизнес-логике.
3. Единый append-only `events` log на всю систему. Не создавай отдельные `*_log` таблицы
   под каждую сущность — сначала проверь, не покрывается ли это `events`.
4. AI/чатбот — вне скоупа. Не предлагай добавить, не закладывай под это таблицы/поля.

## Self-check перед тем, как сказать "готово" (обязательно, каждый раз)

Перед тем как объявить фазу/таску завершённой:

1. Миграции применяются на чистой БД без ошибок (`alembic upgrade head` с нуля).
2. Backend стартует, эндпоинты фазы отвечают (curl/httpie, не только "код написан").
3. Frontend собирается без TS-ошибок (`npm run build` или `tsc --noEmit`).
4. Пройдись по acceptance-критериям фазы в `BUILD_PHASES.md` — реально проверь каждый пункт,
   не полагайся на память о том, что "вроде сделал".
5. Если отклонился от `CRM_SPEC.md` (новая таблица, другое поле, другой flow) — это ДОЛЖНО
   быть записано в `PROGRESS.md` в разделе "Decisions & Assumptions" с коротким обоснованием.
   Молчаливые отклонения от спеки — главная причина, почему схема потом разъезжается.
6. Обнови `PROGRESS.md`: что сделано, что дальше, известные проблемы.

Если пункт 1-3 не проходит — фаза не завершена, не пиши "готово, но с оговоркой".
